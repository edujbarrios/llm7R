# Contributing to llm7R

Thank you for your interest in contributing to **llm7r**. Contributions of all kinds are welcome, including bug reports, feature requests, documentation improvements, and code contributions.

This document provides guidelines to help ensure a smooth and collaborative contribution process.

Please note that all ownership of the API belongs to its respective owner, the contributions will be just to improve this third party client.
---

## Code of Conduct

All contributors are expected to adhere to a respectful and inclusive code of conduct. Be professional, constructive, and considerate in all interactions.

---

## Ways to Contribute

### Report Bugs

If you encounter a bug, please open an issue and include:

- A clear and descriptive title  
- Steps to reproduce the issue  
- Expected behavior  
- Actual behavior  
- Error messages or logs (if any)  
- Your environment details (R version and operating system)  

---

### Suggest Enhancements

Feature requests and improvements are welcome. When submitting a suggestion, please include:

- A clear description of the proposed feature  
- The motivation and use case  
- Minimal reproducible examples, if applicable  

---

### Improve Documentation

Documentation improvements are highly appreciated. You can contribute by:

- Fixing typos or improving clarity  
- Adding examples  
- Expanding function descriptions  
- Improving structure and readability
